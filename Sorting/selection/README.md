# Selection Sort Notes

An elderly sorting algorithm that emerged in the early days of computer science during the 1950s where it was considered one of the foundational, early methods.

The algorithm operates by dividing the dataset into two parts: Sorted on the left and Unsorted on the right. It them repeatedly selects the smallest element form the unsorted part.

**Origin**
- Unknown inventor
- Got its name from its fundamental step: repeatedly selecting elements.

**Time & Space Complexity**

| Case | Time Complexity | Space Complexity |
| :--- | :---: | :---: |
| **Best Case** | $O(n^2)$ | $O(1)$ |
| **Average Case** | $O(n^2)$ | $O(1)$ |
| **Worst Case** | $O(n^2)$ | $O(1)$ |
